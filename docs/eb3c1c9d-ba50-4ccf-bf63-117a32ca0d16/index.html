High Availability Mode Replication

test

Setup Datastore for Virtual Machine​

  • We use ZFS-RAIDZ (software-defined storage)
  • There are 2 data disks on every PVE server

  • Create ZFS on PVE-01 and tick

  • Create ZFS on PVE-02, PVE-03 but do not tick “Add Storage”

  • Add DATASTORE to all PVE servers

  • Now We have the same storage named DATASTORE on all PVE servers

2. HA Mode Configuration​

2.1 Setup New Virtual Machine​

  • Choose the ISO file that has been uploaded before

  • Keep default setting

  • Create VM Disk (if running window we can switch to SATA)

  • Set the number of core CPU and RAM capacity

  • Set Network Interface Card type, Mac Address and NIC connect to “Bridge” (Switch)

  • Confirm to create a VM

  • VM was created and we can modify VM hardware, set boot order, and auto start,…

  • Start the VM and set up the OS as running on bare-metal

2.2 Configuration for VM Replication​

  • VM are running on PVE-01 and should be replicate to PVE-02, PVE-03

  • Set time for replicating, do the same to PVE-03

  • We can now start syncing data

  • Now all PVE have VM Disk

2.3 Set HA mode for VM​

  • Create an HA group that Vm can migrate to these server in the group

  • Enable HA mode for VM

2.4 Test HA Mode​

  • VM running on PVE-02

  • Power off PVE-02

  • Can not ping to VM

  • Cluster take 3-5 minutes to make sure PVE-02 is down
  • VM will migrate to PVE-01 automatically and start successfuly
